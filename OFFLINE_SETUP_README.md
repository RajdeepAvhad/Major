# EatRight ML Improvements - Offline Setup (No API Required)

## 🎯 Overview

This guide shows how to expand the EatRight dataset **without using the USDA API**, which is geo-blocked for Indian IPs. Instead, we use a pre-downloaded CSV file.

---

## 📥 Step 1: Download USDA CSV

### Option A: Direct Download (Recommended)

1. Visit: https://fdc.nal.usda.gov/download-datasets.html
2. Find **"Foundation Foods"** section
3. Click **"Download"** (CSV format)
4. Extract the ZIP file
5. Copy the CSV to: `static/data/FoodData_Central_foundation_food.csv`

**File size**: ~50-100 MB  
**Format**: CSV  
**No API key needed**: ✅

### Option B: Alternative Sources

If the official site is slow or blocked:
- Use a VPN to access the USDA site
- Ask a colleague outside India to download
- Use archived versions from data repositories

---

## 🚀 Step 2: Run the Parser

Once you have the CSV file in place:

```bash
python scripts/parse_usda_local.py
```

**What it does**:
- ✅ Reads local CSV (no internet required)
- ✅ Filters relevant food categories
- ✅ Maps nutrients to EatRight schema
- ✅ Sets meal flags automatically
- ✅ Outputs `food_expanded.csv`

**Expected output**:
```
USDA Local CSV Parser
============================================================
✓ Loaded CSV with 5000+ rows
✓ Filtered: 5000 → 400 foods
✓ Extracted 400 valid food items
✓ Saved to: static/data/food_expanded.csv
```

---

## 🔄 Step 3: Merge Datasets

```bash
python scripts/merge_datasets.py
```

**What it does**:
- Merges `food.csv` + `food_expanded.csv` + `indian_foods.csv`
- Removes duplicates
- Normalizes nutrient values
- Outputs `food_master.csv` (500+ foods)

---

## 🗄️ Step 4: Run Migrations

```bash
python manage.py migrate
```

Applies the Food model updates (veg, glycemic_index, cuisine fields).

---

## 📊 Step 5: Evaluate Model (Optional)

```bash
python scripts/evaluate_model.py
```

Runs 5-fold cross-validation and saves metrics to `model_eval.json`.

## 🔍 Troubleshooting

### Issue: CSV file not found

**Error**: `❌ ERROR: USDA CSV file not found`

**Solution**:
```bash
# Check file exists
ls static\data\FoodData_Central_foundation_food.csv

# If not, download from USDA website
# See USDA_DOWNLOAD_GUIDE.md
```

### Issue: No foods extracted

**Error**: `❌ No valid foods extracted`

**Solution**:
- Check CSV file is not corrupted
- Try re-downloading from USDA
- Check CSV has correct format

### Issue: Encoding error

**Error**: `UnicodeDecodeError`

**Solution**:
- Script tries multiple encodings automatically
- If still fails, open CSV in Excel and save as UTF-8
